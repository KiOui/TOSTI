Dynamic client registration

RFC 7591 endpoint for self-registering OAuth clients

This URL is the RFC 7591 dynamic client registration endpoint for TOSTI. It is meant to be consumed by an OAuth-aware client that wants to register itself programmatically — not opened in a browser. Send a POST with a JSON payload describing your client; you'll get back a client_id for a public, authorization-code-only client with PKCE required.

If you're a regular user, you probably arrived here by accident; the only thing humans do at this URL is read this page.

If you're building an integration, see the OAuth integration guide for the full details of the request shape, what the server provisions, the rate-limit, and which scopes you can ask for. Existing applications you've registered yourself appear under My account → API credentials.
